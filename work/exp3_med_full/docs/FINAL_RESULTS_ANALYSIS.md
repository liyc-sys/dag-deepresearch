## 失败原因深度分析

### DRB2为什么失败？

#### 1. **任务需求不匹配** ❌

**DRB2要求**：
- ✓ 详细列举23个具体研究
- ✓ 创建包含8列的汇总表格
- ✓ 每个研究的具体数据（国家、设计、成分、血压基线等）
- ✓ 精确的亚组分析数据（年龄、样本量、研究时长）

**Report框架输出**：
- ✗ 综合性文献综述（提取共性规律）
- ✗ 没有详细的研究列表
- ✗ 没有表格化呈现
- ✗ 侧重整体结论而非逐项列举

#### 2. **案例对比：成功 vs 失败**

**✅ 成功案例 task26 (33/41)**：
- Info Recall: 22/26 (84.6%) - 基本覆盖了所需信息点
- Analysis: 11/11 (100%) - 分析完整
- Presentation: 0/4 (0%) - **仍缺少表格格式**
- 报告长度: 14,051字符

**❌ 失败案例 task75 (5/41)**：
- Info Recall: 1/26 (3.8%) - 只提到1个研究（SSaSS）
- Analysis: 3/11 (27.3%) - 分析浅显
- Presentation: 1/4 (25%) - 没有要求的表格
- 报告长度: 23,040字符（更长但无用）

**关键洞察**：即使是唯一通过的task26，也因为缺少表格（Presentation=0）而只是勉强通过！

#### 3. **架构层面的根本矛盾**

| 维度 | DRB2需求 | Report框架设计 | 匹配度 |
|------|----------|---------------|--------|
| 信息粒度 | 细粒度（逐个研究） | 粗粒度（综合分析） | ❌ 不匹配 |
| 呈现方式 | 结构化表格 | 自然语言段落 | ❌ 不匹配 |
| 搜索策略 | 精确检索特定研究 | 广泛搜索综合文献 | ❌ 不匹配 |
| 分析方式 | 逐项核对checklist | 整体性归纳总结 | ❌ 不匹配 |

### 成功模式识别

**Report框架最适合的任务特征**：
1. ✅ **开放式问题** - 无固定答案格式
2. ✅ **深度分析** - 需要综合多角度视角
3. ✅ **整体性评估** - 关注质量而非细节完整性
4. ✅ **自然语言输出** - 段落式报告

**不适合的任务特征**：
1. ❌ **详细列举** - 需要逐项核对具体研究
2. ❌ **结构化表格** - 固定格式要求
3. ❌ **细项checklist** - Rubric式评分
4. ❌ **精确数据点** - 具体数值和参数

## 改进方向（如果要攻克DRB2）

### 方案1: 架构扩展（表格生成能力）

**问题**：Report不生成结构化表格

**解决**：
1. 添加TableAgent专门负责表格章节
2. 使用结构化提示词：
   ```
   Create a table with columns: Study, Country, Design, ...
   Extract data from each RCT found in the search results
   ```
3. 输出格式强制为Markdown表格

**预期提升**：Presentation 0/4 → 3/4

---

### 方案2: 搜索策略调整（精确研究检索）

**问题**：Report搜索策略是广泛综合，而非精确列举

**解决**：
1. 添加"Study Enumeration"步骤
2. 搜索查询改为：
   ```
   "salt substitute RCT list 2022"
   "randomized controlled trial salt substitute cardiovascular"
   ```
3. 每个搜索结果提取研究名称和数据

**预期提升**：Info Recall 15% → 60%

---

### 方案3: Prompt优化（强调细节）

**问题**：现有prompt强调综合分析，而非详细列举

**解决**：
在Section Planning中添加：
```
For literature review sections:
- List ALL relevant studies found (minimum 20 studies)
- Extract specific data points: baseline BP, composition, duration, etc.
- Create summary tables with required columns
- Preserve exact numerical values
```

**预期提升**：整体得分 16% → 50%

---

### 方案4: 评估是否值得改进

**成本**：
- 架构调整: ~2周开发

**收益**：

- 论文叙事: 从"专家框架"变成"通用框架"（失去特色）

**建议**：
